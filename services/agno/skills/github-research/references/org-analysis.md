# GitHub Organization Analysis Checklist

## Quick Assessment (5 minutes)
1. How many public repos? (< 10 = focused, > 50 = platform play)
2. Most starred repo? (this is their flagship)
3. Total contributors across repos? (team size indicator)
4. Commit frequency on main repos? (velocity indicator)

## Deep Assessment (30 minutes)
1. Read the top 3 repos' READMEs
2. Check the last 10 merged PRs on the flagship repo
3. Read the last 3 release notes
4. Check open issues sorted by reactions (community priorities)
5. Look at the CONTRIBUTING.md (how open is the project?)
6. Check if there's a roadmap (public project board, milestones)

## Red Flags
- No releases in 6+ months
- Issues with no responses from maintainers
- PRs from external contributors sitting unreviewed
- README says "coming soon" for core features
- License change in recent history
